Stakeholder Requirements (SR)

The following stakeholder requirements define the needs and expectations of the primary actors of the QFit Universal Workout Tracker system.

SR-1: User Requirements

• SR1.1 Users shall be able to track workouts from any location.
• SR1.2 Users shall be able to view progress charts and performance history.
• SR1.3 Users shall be able to set, edit, and track personal fitness goals.
• SR1.4 Users shall be able to manage their account settings (registration, login, profile updates, password reset).
• SR1.5 Users shall be able to sync their activity and biometric data from Google Fit or Apple Health.
• SR1.6 Users shall expect accurate activity data, without duplicate entries between manual logs and API-synced data.
• SR1.7 Users shall be able to manually refresh or disconnect third-party API connections at any time.
• SR1.8 Users shall be notified when biometric syncing fails or API access is revoked.

SR-2: Admin Requirements

• SR2.1 Admins shall be able to view all user profiles.
• SR2.2 Admins shall be able to activate or deactivate user accounts.
• SR2.3 Admins shall be able to reset user passwords.
• SR2.4 Admins shall be able to manage user roles and permissions.

SR-3: System Requirements

• SR3.1 System shall authenticate users securely and ensure protected access.
• SR3.2 System shall store and maintain user workout data, biometric data, and account data securely.
• SR3.3 System shall generate analytics, progress charts, and performance trends.
• SR3.4 System shall reliably sync, process, and update biometric data from Google Fit / Apple Health.
• SR3.5 System shall prevent data conflicts such as double-counting between manual logs and external API data.
